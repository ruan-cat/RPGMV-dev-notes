## git 合并代码普遍冲突的情况

1. 要及时拉取代码，合并主分支。每天都要做的。
2. 不要让自己的分支落后太多。落后太多提交的分支，是不能合并的。
3. 合并更改时，不应该出现任何空合并。空合并就相当于删除全部代码了。
4. 不要一次性提交太多东西，每做完一个页面，就提交一个页面。不要搞得一次性提交一大堆东西。
