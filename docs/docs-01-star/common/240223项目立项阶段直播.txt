	可行性分析:
		x 社会可行性
		x 经济可行性
		
		技术可行性:
			这个项目 咱们技术能不能做的来
			
			1. 用到哪些技术:
				比如: 用结构体吗?  要不要分文件开发? 界面是用easyx还是swing还是控制台?
				
				注意:
					如果 打算用的这个技术 只有部分代码会涉及到
						需要有一半人掌握
						比如: easyx swing 文件流 数据库
					如果 打算用的技术 所有代码都会涉及到
						比如: 函数, 面向对象, 结构体, git, spring
					
			2. 列举核心功能:
				五子棋: 
					核心功能: 
						胜利判断, 落子(轮流), 展示棋盘和棋子
						
					拓展功能: 
						联机, 悔棋, 人机, 残局, 皮肤, 商城, 氪金, 战令, 万宁五子棋, 排位
				
				贪吃蛇:
					核心功能: 
						定时移动, 移动判断(吃食物 长度+1 生成新食物, 撞墙,撞自己, 空地), 修改方向
						展示蛇和地图
				
					拓展功能: 
						复活 积分 不同地图 xxxxxxxxx
			
			3. 考虑是否能实现:
				1. 数据的设计: 根据核心功能列举 想想要用到哪些数据
					要求用代码描述
					1. 变量声明(类型 变量名)
					2. 详细的注释(变量含义, 值的含义)
					
					五子棋
						胜利判断, 落子(轮流), 展示棋盘和棋子
						
						// 表示回合数  奇数表示白子  偶数表示黑子
						int flag;
						
						/*
							表示棋盘
								map[i][j]表示棋盘坐标(i,j)的值
								0表示空地
								1表示黑子
								2表示白子
							如: map[3][6] = 1  表示坐标(3,6)是黑子
						*/
						int map[19][19];
					
					
					贪吃蛇: 
						定时移动, 
						移动判断(吃食物 长度+1 生成新食物, 撞墙,撞自己, 空地), 
						修改方向
						展示蛇和地图
						
						/*
							地图: 
								食物 空地 墙
							0表示空地
							1表示墙
							2表示食物
							如: map[3][6] = 2  表示坐标(3,6)是食物
							
						*/
						int map[H][W];
						
						/*
							蛇身坐标
							
							s[i][j]表示蛇的第i节身体的j坐标
							比如: s[2][0] = 3  s[2][1] = 5;
								表示蛇的第[2]节身体 坐标是(3,5)
						*/
						int s[H*W][2] = {
							{2,6},
							{2,7},
							{2,8},
							{2,9},
							{3,9},
						}
						
						// 表示蛇的长度  len=3 表示当前蛇长度为3
						int len;
						
						/*
							蛇的当前方向 'w' 'a' 's' 'd'
						*/
						char dir;
						
						
				2. 逻辑的描述
					文字描述即可
					描述核心功能是怎么通过数据来展示/修改的
						胜利判断:
							根据玩家落子点的坐标 判断周围是否连成五子
							如果连成 游戏结束 根据flag的值判断是否获得胜利
							
						落子(轮流):
							根据玩家落子点 判断这个map上的位置有没有棋子了
							如果有 落子失败
							否则 落子成功 map对应位置 赋值为对应的颜色
						
					
			整理:
				1. 考虑用哪些技术: 比如界面是控制台 easyx swing  要不要用结构体 文件流
				2. 考虑这个项目 有哪些核心功能
				3. 根据核心功能 整理数据的设计和逻辑的描述
					数据设计要求代码描述
					逻辑描述用文字即可
